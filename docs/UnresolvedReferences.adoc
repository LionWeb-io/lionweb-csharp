= Unresolved References

The code generator supports two ways of dealing with unresolved references, to be set in the `GeneratorConfig`:

Throws::
Any access to an unresolved reference throws an `UnresolvedReferenceException`.
If the reference contains any unresolved target, `TryGet()` returns `false`.

ReturnAsNull::
Unresolved references are returned as `null`.
`TryGet()` acts as if `null` is a regular value, i.e. as soon as a reference is present, no matter whether it can be resolved, `TryGet()` returns `true`.
+
This also changes the type of the generated C# property from `ReferenceType` to `ReferenceType?` (for _required single references_) and from `List<ReferenceType>` to `List<ReferenceType?>` for _multiple references_, both _single_ and _multi_.
